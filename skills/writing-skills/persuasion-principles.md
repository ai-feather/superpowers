# 技能设计中的说服原则

## 概览

LLM 对说服原则的反应与人类相同。理解这套心理学有助于你设计出更有效的技能——不是为了操纵，而是为了确保关键实践即使在压力下也能被遵守。

**研究基础：** Meincke 等人（2025）用 N=28,000 次 AI 对话测试了 7 条说服原则。说服技巧把顺从率提高了一倍多（33% → 72%，p < .001）。

## 七条原则

### 1. 权威（Authority）
**它是什么：** 对专业能力、资质或权威来源的服从。

**在技能中如何起作用：**
- 祈使语气："你必须（YOU MUST）"、"绝不（Never）"、"永远（Always）"
- 不可协商的框架："没有例外"
- 消除决策疲劳和自我合理化

**何时使用：**
- 强制纪律的技能（TDD、验证要求）
- 安全关键实践
- 既定的最佳实践

**示例：**
```markdown
✅ 先写代码再写测试？删掉它。从头来。没有例外。
❌ 在可行时考虑先写测试。
```

### 2. 承诺（Commitment）
**它是什么：** 与先前行动、声明或公开宣告保持一致。

**在技能中如何起作用：**
- 要求声明："声明技能使用"
- 强制显式选择："在 A、B 或 C 中选择"
- 使用追踪：用待办来跟踪检查清单

**何时使用：**
- 确保技能被真正执行
- 多步流程
- 问责机制

**示例：**
```markdown
✅ 当你找到一个技能时，你必须声明："我正在使用 [技能名]"
❌ 考虑让搭档知道你在用哪个技能。
```

### 3. 稀缺（Scarcity）
**它是什么：** 因时间限制或有限可用性而产生的紧迫感。

**在技能中如何起作用：**
- 有时限的要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**何时使用：**
- 即时验证要求
- 对时间敏感的工作流
- 防止"我待会儿再做"

**示例：**
```markdown
✅ 完成一个任务后，立即请求代码评审，然后再继续。
❌ 你可以在方便时评审代码。
```

### 4. 社会认同（Social Proof）
**它是什么：** 顺从他人的做法或被视为常态的事物。

**在技能中如何起作用：**
- 普遍模式："每一次（Every time）"、"总是（Always）"
- 失败模式："没有 Y 的 X = 失败"
- 建立规范

**何时使用：**
- 记录普遍实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有待办追踪的检查清单 = 步骤被跳过。每一次都是。
❌ 有些人觉得检查清单配个待办列表会有帮助。
```

### 5. 归属（Unity）
**它是什么：** 共同的身份认同、"我们感"、群体内的归属感。

**在技能中如何起作用：**
- 协作语言："我们的代码库"、"我们是同事"
- 共同目标："我们都想要质量"

**何时使用：**
- 协作式工作流
- 建立团队文化
- 非等级制实践

**示例：**
```markdown
✅ 我们是一起工作的同事。我需要你诚实的技术判断。
❌ 我错了的话你大概应该告诉我。
```

### 6. 互惠（Reciprocity）
**它是什么：** 回报所受恩惠的义务。

**如何起作用：**
- 谨慎使用——会显得有操纵性
- 在技能中很少需要

**何时避免：**
- 几乎总是避免（其他原则更有效）

### 7. 喜爱（Liking）
**它是什么：** 倾向于与我们喜欢的人合作。

**如何起作用：**
- **不要用于强制顺从**
- 与诚实反馈文化相冲突
- 制造阿谀奉承

**何时避免：**
- 在强制纪律时永远避免

## 按技能类型的原则组合

| 技能类型 | 使用 | 避免 |
|------------|-----|-------|
| 强制纪律 | 权威 + 承诺 + 社会认同 | 喜爱、互惠 |
| 指导/技巧 | 适度权威 + 归属 | 重度权威 |
| 协作 | 归属 + 承诺 | 权威、喜爱 |
| 参考 | 仅需清晰 | 所有说服手段 |

## 为什么有效：心理学

**明确的界线减少自我合理化：**
- "你必须"消除决策疲劳
- 绝对化语言消除"这是不是一个例外？"的问题
- 显式的反合理化反制关闭特定的漏洞

**执行意图创造自动化行为：**
- 清晰的触发条件 + 要求的动作 = 自动执行
- "当 X 时，做 Y"比"一般性地做 Y"更有效
- 降低顺从的认知负荷

**LLM 是类人的（parahuman）：**
- 在包含这些模式的人类文本上训练
- 训练数据中权威语言先于顺从出现
- 承诺序列（声明 → 行动）被频繁建模
- 社会认同模式（每个人都做 X）建立规范

## 伦理使用

**正当的：**
- 确保关键实践被遵守
- 创建有效的文档
- 防止可预见的失败

**不正当的：**
- 为个人利益而操纵
- 制造虚假紧迫感
- 基于内疚的顺从

**判断标准：** 如果用户完全理解这套技巧，它是否仍然服务于用户的真实利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七条说服原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 用 N=28,000 次 LLM 对话测试了 7 条原则
- 顺从率随说服技巧从 33% 提升到 72%
- 权威、承诺、稀缺最有效
- 验证了 LLM 行为的类人模型

## 快速参考

设计一个技能时，问自己：

1. **它是什么类型？**（强制纪律 vs. 指导 vs. 参考）
2. **我想改变什么行为？**
3. **哪些原则适用？**（强制纪律通常是权威 + 承诺）
4. **我是不是组合得太多了？**（不要七条全用）
5. **这是否符合伦理？**（是否服务于用户的真实利益？）
